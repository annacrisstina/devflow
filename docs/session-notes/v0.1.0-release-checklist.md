## Release steps (founder, after this PR merges)

0. **Pre-tag gate:** re-run `pnpm verify`, `pnpm e2e` and the stranger test (fresh clone → [self-hosting.md](../self-hosting.md) followed literally → `pnpm install && pnpm build` → `pnpm demo:seed` → populated dashboard) on a healthy Docker — the machine's Docker Desktop was failing when this checklist was finalized.
1. Merge the PR; confirm CI green on `main` (the dogfood artifact appears on the run).
2. `git tag -a v0.1.0 -m "DevFlow v0.1.0"` on the merge commit; `git push origin v0.1.0`.
3. Create the GitHub Release from the tag; body = the 0.1.0 section of [CHANGELOG.md](../../CHANGELOG.md).
4. Real-GitHub pass (standing since M4): reconfigure the App (§3b), log in, claim, install on this repo + `devflow-demo-flaky`, watch ingestion live.
5. Record the demo video ([storyboard](../../scripts/demo/README.md)).
6. After the tag: work the Dependabot majors queue, individually (D-M6-6).
